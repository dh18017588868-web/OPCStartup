# OPC创业七步法

> 基于SYB创业十步调整，专为一人公司/超级个体设计
> 原SYB十步 → 精简为更适合个体创业的七步

---

## 与SYB十步的对照

| SYB十步（传统） | → | OPC七步（个体） | 核心差异 |
|----------------|---|-----------------|----------|
| 1. 创业者评价 | → | **1. 自我评估** | 增加技能/资源优势分析 |
| 2. 企业构思 | → | **2. 赛道选择** | 强调小众刚需 |
| 3. 市场评估 | → | **3. 客户验证** | 强调先验后做 |
| 4. 人员组织 | → | **4. 资源整合** | 用AI/外包代替招人 |
| 5. 法律形态 | → | **5. 合规准备** | 简化（个人/个体户） |
| 6. 法律责任 | → | **（并入5）** | 合并简化 |
| 7. 启动资金 | → | **6. 成本核算** | 强调轻资产 |
| 8. 利润计划 | → | **（并入6）** | 合并为盈亏平衡 |
| 9. 可行性分析 | → | **7. 启动执行** | 强调快速MVP |
| 10. 开办企业 | → | **（并入7）** | 强调持续迭代 |

### 第3步：客户验证 🎯

**目标**：回答"真的有人愿意付费吗"

---

#### 📋 验证流程（1-2周）

```
Day 1-2: 找到10个目标用户
    ├── 在哪里：社群/论坛/竞品用户群
    └── 怎么找：Cold Email/付费咨询

Day 3-4: 访谈5个用户
    ├── 问题1：这个痛点对你影响多大？
    ├── 问题2：你目前怎么解决的？
    └── 问题3：你愿意为更好的方案付多少钱？

Day 5-7: 验证付费意愿
    ├── 众筹验证：先收钱再开发
    ├── 预售验证：发意向表单
    └── 访谈验证：直接问"如果上线你会买吗"
```

**验证成功标准**：
- 10人中≥3人表示愿意付费
- 有人愿意付定金

#### ⚡ 快速验证方法

**24小时验证法**：
1. 写一个简易落地页（Notion/Grayscale）
2. 发到社交媒体/社群
3. 看有多少人感兴趣/询问
4. 有10+人询问就算验证通过

**预售验证法**：
1. 写出产品介绍
2. 直接收款（微信/支付宝）
3. 收到钱=验证成功
4. 没收到=验证失败

#### 📋 阶段与形态选择

| 阶段 | 推荐形态 | 年成本 | 复杂度 | 适用场景 |
|------|----------|--------|--------|----------|
| **验证期** | 个人 | 0 | 无 | 0-1万/月 |

| **稳定期** | 公司 | ~5000元 | 较高 | 30万+/月 |

---

#### 🏢 形态对比

| 形态 | 优点 | 缺点 | 税率参考 |
|------|------|------|----------|
| **个人** | 0成本、灵活 | 无法开票、风险无限 | 劳务报酬20% |

| **工作室** | 品牌好、可开票 | 成本高 | 核定征收1-3% |
| **有限责任公司** | 风险隔离、品牌 | 成本高、复杂 | 25% |

## ❓ 常见问题

**Q：一定要按顺序吗？**
A：建议按顺序，但1-2步可以并行思考后再确认赛道

**Q：时间紧张怎么办？**
A：每步都有"快速版"，可以压缩到1-2天完成

**Q：失败了怎么办？**
A：OPC的优势就是低成本，随时可以调整方向或暂停

**Q：没有产品经验怎么办？**
A：先从副业/兼职开始，用主业收入cover风险

**Q：需要多少钱？**
A：轻资产模式500元以内即可启动

**Q：中途卡住了怎么办？**
A：告诉我卡在哪里，我会针对性帮助你
