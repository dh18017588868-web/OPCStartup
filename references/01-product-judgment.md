# 产品判断核心方法论

> 核心问题：这个想法靠谱吗？卖给谁？怎么赚钱？

## 一、判断框架

### 1.1 四个核心问题

回答以下四个问题，形成判断：

| 问题 | 必须回答 |
|------|----------|
| **卖给谁** | 目标用户是谁？具体到什么人群画像 |
| **怎么赚** | 收费模式是什么？一次性/订阅/广告？ |
| **为什么买** | 用户为什么选你而不是别人？替代方案是什么？ |
| **第一单** | 第一单怎么来？谁会是第一个付费的人？ |

### 1.2 判断标准

**靠谱 = 能回答这四个问题，且答案不自相矛盾**

**不靠谱的特征**：
- 目标用户泛泛"所有需要 X 的人"
- 商业模式模糊"先做起来再说"
- 没有替代方案思考"没有竞品"
- 第一单路径不清"等用户自然来"

## 二、分析维度

### 2.1 市场需求分析

| 维度 | 问题 |
|------|------|
| **真实需求** | 用户真的需要这个吗？还是你觉得需要？ |
| **需求强度** | 有多痛？不买会怎样？ |
| **付费意愿** | 用户历史上为什么付费？付过多少？ |
| **市场阶段** | 早期/成长期/成熟期/衰退期？ |

### 2.2 竞争分析

| 维度 | 问题 |
|------|------|
| **直接竞品** | 谁在做一模一样的事？ |
| **替代方案** | 用户不选你，可以选什么？ |
| **竞争壁垒** | 你能做什么别人做不到的？ |
| **差异化** | 有什么不一样？为什么这个不一样重要？ |

### 2.3 商业可行性

| 维度 | 问题 |
|------|------|
| **定价** | 准备卖多少钱？ |
| **毛利** | 毛利率多少？能不能cover获客成本？ |
| **获客** | 从哪里获客？成本多少？ |
| **交付** | 需要投入多少时间交付？ |
| **复购** | 用户会复购吗？还是一锤子买卖？ |

### 2.4 一人公司适配性

| 维度 | 问题 |
|------|------|
| **时间投入** | 一个人能否完成交付？ |
| **技能要求** | 需要哪些技能？一个人能具备吗？ |
| **可扩展性** | 能否用 AI 工具放大产能？ |
| **边际成本** | 每多一个用户，需要额外投入多少？ |

## 四、常见问题

### 4.1 "我没有竞品"

**回答**：没有竞品通常不是好事，说明：
- 可能市场需求不存在
- 可能市场太小没人愿意做
- 可能你还没找到真正的替代方案

**正确姿势**：找到用户的"替代方案"，哪怕是人工做、土办法。

### 4.2 "先做 MVP 试试"

**回答**：MVP 不是不管市场，而是用最小成本验证核心假设。

**正确姿势**：先说清楚验证什么：
- 验证需求真实存在？
- 验证用户愿意付费？
- 验证能找到目标用户？

### 4.3 "目标用户是所有人"

**回答**：卖给所有人 = 卖不出去。

**正确姿势**：找到最核心的 1000 人，他们是谁？有什么共同特征？

## 五、判断示例

### 示例 1：AI 提示词工具

**输入**：我想做一个 AI 提示词分享网站

**判断**：

| 问题 | 答案 |
|------|------|
| 卖给谁 | 想要用好 AI 工具的人，主要是运营、产品、开发者 |
| 怎么赚 | 免费分享 + 付费高级提示词，$9.9-49.9 |
| 为什么买 | 节省学习时间，获得高质量可直接用的提示词 |
| 第一单 | 在小红书/知乎发布优质提示词，引流到付费区 |

**结论**：可行，但需要验证内容获客能力

### 示例 2：做一款AI聊天App

**输入**：我想做一个 AI 聊天App，类似 ChatGPT

**判断**：

| 问题 | 答案 |
|------|------|
| 卖给谁 | 还没想清楚 |
| 怎么赚 | 还没想清楚 |
| 为什么买 | 比 ChatGPT 更好？还没想清楚 |
| 第一单 | 还没想清楚 |

**结论**：不靠谱，需要先回答核心问题
