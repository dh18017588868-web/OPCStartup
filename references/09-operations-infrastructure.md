# 一人公司运营基础设施

> 核心问题：产品做出来了，怎么收钱？怎么触达用户？怎么管理客户？

---

## 一、支付解决方案

### 1.1 国内支付方案对比

| 方案 | 费率 | 开通条件 | 适用场景 |
|------|------|----------|----------|
| **微信支付（商户）** | 0.6% | 需要营业执照 | 正式商业运营 |
| **支付宝（商户）** | 0.6% | 需要营业执照 | 正式商业运营 |
| **小微商户** | 0.38% | 无营业执照限制 | 个人起步阶段 |
| **面包多 Pay** | 0.1%+2元 | 个人即可 | 数字产品销售 |
| **Paddle** | 5%+50¢ | 需企业 | 面向海外 |

### 1.2 一人公司推荐支付路径

**阶段一：个人小微（起步期）**
```
推荐方案：小微商户 + 面包多Pay
- 小微商户：收 ¥1万以下/月的收款
- 面包多：数字产品自动发货
```

**阶段二：个体工商户（成长期）**
```
注册个体户，开通正式商户号
- 微信支付：0.6%
- 支付宝：0.6%
- 可开对公账户
```

**阶段三：有限责任公司（规模化）**
```
注册公司，开通企业支付
- 正规化运营
- 可开发票
- 便于融资
```

### 1.3 支付接入 Checklist

```
□ 选择支付方案
□ 准备资质材料（身份证/营业执照）
□ 注册商户号
□ 对接支付接口（或使用现成工具）
□ 测试支付流程
□ 设置退款/售后流程
□ 考虑税费预留（10-15%）
```

## 二、消息触达方案

### 2.1 一人公司消息触达痛点

| 痛点 | 解决方案 |
|------|----------|
| 加微信被限制 | 企业微信/微信小号 |
| 消息回复不过来 | 微信机器人自动回复 |
| 群消息管不过来 | 社群机器人管理 |
| 私域沉淀困难 | 公众号 + 企微 + 个微 |

### 2.2 微信生态触达方案

| 方案 | 工具 | 适用场景 |
|------|------|----------|
| **企业微信** | 官方 | 客户管理、群运营 |
| **个微 + 工具** | Wechaty、微信机器人 | 自动回复、养号 |
| **公众号** | 官方 | 内容沉淀、推送 |
| **视频号** | 官方 | 直播带货、内容分发 |

### 2.3 消息触达工具推荐

| 用途 | 工具 | 说明 |
|------|------|------|
| **微信自动回复** | Wechaty | 可实现关键词回复 |
| **社群管理** | 群打卡机器人 | 促活、管理 |
| **邮件营销** | Mailchimp、SendGrid | 海外用户触达 |
| **短信通知** | 阿里云短信 | 重要通知提醒 |

### 2.4 私域沉淀 SOP

```
Step 1: 引流入口
├── 小红书/知乎简介：加微信领取_____
├── 公众号菜单：扫码添加
└── 包裹卡：好评后引导

Step 2: 承接标准化
├── 欢迎语：说明能提供什么价值
├── 标签：来源渠道 + 兴趣标签
└── 自动拉群：同好社群

Step 3: 价值输出
├── 每日/每周固定内容输出
├── 个性化互动（生日/节日）
└── 限时活动激活

Step 4: 转化路径
├── 朋友圈种草
├── 社群促销
└── 一对一跟进
```

## 六、合规与法律

### 6.1 一人公司常见合规问题

| 场景 | 合规要求 | 建议 |
|------|----------|------|
| **个人收款** | 大额收款风险 | 申请商户号 |
| **知识付费** | 需营业执照 | 注册个体户 |
| **内容创作** | 版权、广告合规 | 注意素材版权 |
| **用户数据** | 个人信息保护 | 遵守 GDPR/个保法 |
| **跨境收款** | 外汇管制 | 使用 Paddle/Stripe |

### 6.2 公司注册建议

| 类型 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **个体工商户** | 简单、低税 | 无限责任 | 初期验证 |
| **个人独资企业** | 税负低 | 无限责任 | 小规模运营 |
| **有限责任公司** | 风险隔离 | 成本稍高 | 规模化运营 |

### 6.3 合规 Checklist

```
□ 营业执照（个体户/公司）
□ 税务登记
□ 支付商户号
□ 域名备案（国内服务器）
□ 用户隐私协议
□ 服务条款
□ 发票开具能力
```
