# 市场调研与需求验证

> 核心问题：这个市场需求真实吗？有多少人需要？愿意付多少钱？

---

## 一、市场调研框架

### 1.1 四层需求验证

| 层级 | 验证问题 | 方法 |
|------|----------|------|
| **需求存在** | 用户真的遇到这个问题吗？ | 用户访谈、评论区挖掘 |
| **需求强度** | 问题有多痛？不解决会怎样？ | 1-10 痛点评分 |
| **付费意愿** | 用户历史上为类似问题付过钱吗？ | 调研历史付费行为 |
| **市场规模** | 目标用户群体够大吗？ | TAM/SAM/SOM 分析 |

### 1.2 一人公司市场选择标准

**小市场原则**：选择年收入 1000 万-5000 万，巨头看不上的市场

| 维度 | 推荐 | 避免 |
|------|------|------|
| **市场规模** | TAM 1000万-10亿 | TAM 过小（难规模化）或过大（巨头竞争） |
| **竞争格局** | 没有头部玩家 | 已形成巨头垄断 |
| **用户需求** | 分散且个性化 | 标准化、巨头可替代 |
| **获客难度** | 可以通过内容获客 | 纯投放才能获客 |

## 二、需求验证方法

### 2.1 7 日冷启动测试法

**适用场景**：正式开发产品前，快速验证需求真实性

```
Day 1-2: 准备
- 制作一个"假"产品页面（Notion/Canva）
- 写清楚解决什么问题、卖多少钱
- 准备好微信/邮箱联系方式

Day 3-5: 获取反馈
- 在目标用户聚集地发布（小红书/知乎/微信群）
- 观察：有多少人询问？有多少人留联系方式？
- 统计：咨询转化率

Day 6-7: 分析结论
- 超过 10% 咨询转化 → 需求真实
- 低于 5% 咨询转化 → 需求存疑
- 0 咨询 → 需求不存在
```

### 2.2 MVP 测试三种变体

| 方法 | 操作 | 适用场景 |
|------|------|----------|
| **纸质原型验证** | 用 Figma/Canva 做高保真图，在社群调研 | 验证功能需求 |
| **虚拟店铺测试** | 闲鱼/小红书挂"即将上线"，观察咨询量 | 验证付费意愿 |
| **预购式需求采集** | 发布"即将涨价"公告，看多少人提前付款 | 验证真实验需求 |

### 2.3 验证 Checklist

```
□ 用户说"确实需要"——但他说的是真的吗？
□ 用户愿意留联系方式吗？（留 = 真需求）
□ 用户问"什么时候上线"吗？（问 = 真需求）
□ 用户愿意付定金吗？（付 = 真需求）
□ 用户会主动推荐给朋友吗？（推荐 = 强需求）
```

## 四、市场规模估算

### 4.1 TAM/SAM/SOM 模型

```
TAM（Total Addressable Market）: 总体市场
├── SAM（Serviceable Addressable Market）: 可服务市场
│   └── SOM（Serviceable Obtainable Market）: 可获得市场
```

**示例：AI 写作工具**

| 层级 | 定义 | 估算 |
|------|------|------|
| TAM | 所有需要写作的人 | 5 亿人 |
| SAM | 需要 AI 辅助写作的人 | 5000 万人 |
| SOM | 一人公司能触达的人 | 1 万人 |

### 4.2 一人公司市场规模参考

| 赛道 | TAM | SOM 可行 | 典型价格 |
|------|-----|----------|----------|
| 效率工具 | 大 | 1-10万用户 | ¥19-99/月 |
| 知识付费 | 中 | 1000-10000付费 | ¥99-1999 |
| 内容创作 | 大 | 5000-5万粉丝 |广告/带货|
| 小微SaaS | 中 | 100-1000企业 | ¥999-9999/年 |

## 五、竞品分析

### 5.1 竞品分析框架

```markdown
## 竞品分析：[竞品名称]

### 基本信息
- 产品定位：_____
- 目标用户：_____
- 定价：_____
- 获客渠道：_____

### 产品分析
- 核心功能：_____
- 优势：_____
- 劣势：_____

### 市场表现
- 预估用户量：_____
- 预估收入：_____
- 增长趋势：_____

### 对我们启示
- 差异化切入点：_____
- 定价参考：_____
- 值得借鉴：_____
```

### 5.2 寻找竞品的方法

| 方法 | 操作 |
|------|------|
| **关键词搜索** | 百度/Google 搜索"_____ 工具/软件/服务" |
| **社群观察** | 加入目标用户社群，看他们用什么 |
| **App Store** | 搜索相关关键词，看排名 |
| **新榜/蝉妈妈** | 查内容平台的热门账号 |
| **Alixmnt** | 查竞品流量来源 |

### 5.3 竞品监控 Checklist

```
□ 直接竞品（做一样的事）
□ 替代方案（用其他方式满足需求）
□ 上游产品（可能切入这个赛道）
□ 潜在竞品（刚融钱、刚上线）
□ 关注竞品更新频率
□ 关注竞品用户评价（吐槽什么）
```

## 七、案例参考

### 案例：AI 提示词库的需求验证

**背景**：想做一个 AI 提示词分享网站

**需求验证过程**：

```
实测方法：在小红书发布收费提示词
- 发布 5 篇笔记
- 获得 2000 浏览
- 私信询问：30 人
- 付费购买：3 人（¥29.9）
- 转化率：10%

结论：
- 需求真实（有人愿意付费）
- 验证路径清晰（小红书内容获客）
- 定价可接受（¥29.9）
- 可扩展方向：会员订阅、定制服务
```

## 八、工具推荐

### 8.1 市场调研工具

| 用途 | 工具 | 说明 |
|------|------|------|
| **关键词分析** | 百度指数、5118 | 搜索热度 |
| **竞品监控** | SimilarWeb、Ahrefs | 流量分析 |
| **社群挖掘** | 微信搜索、新榜 | 热门内容 |
| **问卷调查** | 腾讯问卷、金数据 | 用户调研 |
| **数据分析** | 蝉妈妈、灰豚数据 | 抖音/小红书 |

### 8.2 用户访谈问题模板

```
开场：
- 你是做什么的？用这个产品解决什么问题？

需求验证：
- 这个问题对你影响多大？（1-10）
- 之前怎么解决？不解决会怎样？
- 为这个问题付过钱吗？付了多少？

竞品了解：
- 现在用什么产品？
- 最满意什么？不满意什么？
- 如果有新选择，你会考虑吗？

付费意愿：
- 这个产品卖多少钱你会买？
- 你会因为什么决定购买？
```
