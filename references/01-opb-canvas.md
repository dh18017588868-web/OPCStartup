# 一人企业画布 - 完整填写指南

> 基于 EasyChen 一人企业方法论，在商业模式画布基础上专为一人企业设计

## 画布概述

**核心目的**：将业务规划结构化，避免凭感觉做事

**使用时机**：
- 开始新业务前
- 业务遇到瓶颈时
- 每月复盘时

---

## 8大要素详解

### 1️⃣ 细分客户（Customer Segments）

**问题**：谁是你的客户？

**填写要点**：
- 越具体越好，避免"所有需要 XXX 的人"
- 定位「小众刚需」群体
- 明确客户的身份特征、使用场景、决策方式

**模板**：
```
目标客户：[具体人群，如：独立开发者]
使用场景：[具体场景，如：周末开发小工具]
决策方式：[个人决策/团队决策/老板决策]
付费能力：[高/中/低，愿意为什么付费]
```

**示例**：
- ❌ 大众：需要存储的人
- ✅ 精准：技术博主，需要在自己的博客文章中嵌入代码片段的开发者

---

### 2️⃣ 价值主张（Value Proposition）

**问题**：解决客户的什么问题？提供什么独特价值？

**填写要点**：
- 小众价值主张往往「剑走偏锋」，而非四平八稳
- 如果大众价值主张涵盖了你的内容，就缺乏竞争力
- 必须回答：为什么客户选择你而不是现有方案？

**模板**：
```
核心痛点：[客户最痛的点是什么？]
现有方案：[客户现在用什么？有什么不满？]
你的方案：[你提供的独特解决方案]
差异化：[和现有方案相比的优势]
```

**示例对比**：
| 类别 | 大众方案 | 小众方案（价值主张） |
|------|----------|---------------------|
| 网盘 | 价格、速度 | 隐私、P2P下载、美剧资源整合 |
| 笔记 | 功能全、跨平台 | 纯Markdown、Git版本管理 |
| 建站 | 模板多、SEO好 | 极简、开发者友好、API优先 |

---

### 3️⃣ 渠道通路（Channels）

**问题**：如何接触到目标客户？

**填写要点**：
- **渠道通路是项目最容易失败的地方**
- 可参与客户论坛、加入微信群、深入了解客户交流内容
- 如果找不到解决方案，可回到细分客户重新选择

**模板**：
```
主要渠道：[哪个渠道最有效？]
获客成本：[预估获取一个客户的成本]
转化路径：[客户从知道到付费的路径]
触达频率：[多久触达一次客户？]
```

**常见渠道**：
| 渠道类型 | 适用场景 | 难度 |
|----------|----------|------|
| 垂直社区 | 目标用户聚集的论坛/社群 | 中 |
| 内容营销 | 博客、小红书、B站 | 高但长期有效 |
| 付费投放 | 有一定预算时 | 高 |
| SEO | 长期积累型 | 高 |
| 口碑推荐 | 产品已有用户基础 | 低 |

---

### 4️⃣ 竞争策略（Competitive Strategy）

**问题**：如何在竞争中取胜？

**填写要点**：
- 判定竞争策略是否有效
- 考虑如何发现和扩大结构性优势

**三种核心策略**：

#### 策略一：不竞争策略
- 选择无人竞争的细分市场
- 找到大公司看不上的小机会
- 示例：为大公司服务的小众插件

#### 策略二：结构化优势
- 利用自己的独特背景/技能/资源
- 构建竞争对手难以复制的优势
- 示例：程序员背景做开发者工具

#### 策略三：流量收敛策略
- 在一个点做到极致，形成口碑
- 让客户主动来找你
- 示例：专注一个领域的KOL

---

### 5️⃣ 原有方案与痛点（Problems）

**问题**：客户现在用什么方案？有什么不满？

**填写要点**：
- 仔细思考现有方案执行某项任务时的缺陷
- 识别可提升的点
- 痛点越痛越好

**模板**：
```
客户正在使用：[列出竞品和替代方案]
不满之处：[现有方案的3个主要问题]
严重程度：[小痛/中痛/剧痛]
频率：[偶尔/经常/每天]
```

---

### 6️⃣ 解决方案（Solution）

**问题**：你提供什么方案来解决痛点？

**填写要点**：
- 确保新方案比现有方案更具吸引力
- 不需要完美，只需要好一点点
- 聚焦在一个核心点

**模板**：
```
核心功能：[最核心的1-3个功能]
解决方式：[如何解决痛点]
独特亮点：[有什么特别之处]
```

---

### 7️⃣ 关键业务（Key Activities）

**问题**：需要做什么事情来实现解决方案？

**三个特别注意**：

1. **单点切入**
   - 资源有限，必须集中所有资源于一点
   - 找到最小的可验证切入点

2. **先验后做**
   - 产品开发前先验证市场匹配度
   - 可用MVP、众筹、试播等方式验证

3. **控制周期**
   - 对一人企业至关重要
   - 若周期过长，需求可能已变化
   - 建议：MVP控制在2-4周

**模板**：
```
核心切入点：[最小可验证的产品是什么？]
验证方式：[如何验证需求真实存在？]
开发周期：[预估需要多长时间？]
关键里程碑：[重要的节点是什么？]
```

---

### 8️⃣ 成本、收入和资产评分（Cost, Revenue & Asset Score）

**成本**：
- 资金成本：开发、服务器、推广等
- 时间成本：自己的时间投入
- 规模化成本：增长时的边际成本

**收入**：
- 收入来源：一次性/订阅/增值
- 定价策略
- 毛利率预估

**资产评分（4维度）**：
| 维度 | 问题 | 评分 |
|------|------|------|
| 投入产出比 | 赚的钱值得投入的时间和钱吗？ | 1-5 |
| 风险 | 需求真实吗？有人愿意付费吗？ | 1-5 |
| 持久性 | 这个需求会长期存在吗？ | 1-5 |
| 进入门槛 | 别人容易复制吗？ | 1-5 |

**总分**：12分以下不建议做，12-16分可尝试，17-20分值得全力投入

## 画布使用流程

```
1. 先确定细分客户
      ↓
2. 提出价值主张（针对小众刚需）
      ↓
3. 规划渠道通路（关键环节，决定生死）
      ↓
4. 制定竞争策略
      ↓
5. 分析痛点与解决方案
      ↓
6. 规划关键业务（注意三个重点）
      ↓
7. 计算成本收入，进行资产评分
      ↓
8. 每月使用月报复盘迭代
```

## 使用场景示例

### 场景：我想做 AI 提示词工具

**填写示例**：

| 要素 | 内容 |
|------|------|
| 细分客户 | 运营人员、需要写文案的自媒体人 |
| 价值主张 | 专注于小红书文案的AI提示词模板，解决不会写、写得慢的问题 |
| 渠道通路 | 小红书笔记、微信社群、B站教程 |
| 竞争策略 | 结构化优势 - 运营背景，更懂运营需求 |
| 原有方案 | ChatGPT直接对话、通用提示词网站 |
| 痛点 | 通用提示词不好用、不了解Prompt Engineering |
| 解决方案 | 按场景分类的提示词模板 + 最佳实践指南 |
| 关键业务 | 先做20个高频场景的模板，验证付费意愿 |
| 成本收入 | 开发成本5000，定价49元/模板包，预计月销100+ |
