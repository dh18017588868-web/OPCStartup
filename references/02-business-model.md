# 商业模式设计

> 核心问题：怎么赚钱？赚多少？能否持续？

## 一、商业模式画布

### 1.1 核心要素

| 要素 | 问题 | 一人公司建议 |
|------|------|--------------|
| **价值主张** | 你提供什么价值？ | 越具体越好，不要泛泛"提高效率" |
| **目标用户** | 谁愿意为此付费？ | 越小众越容易建立信任 |
| **渠道触达** | 怎么找到用户？ | 内容获客为主，付费投放为辅 |
| **客户关系** | 如何维护客户？ | 轻运营，用产品代替人工 |
| **收入来源** | 怎么收费？ | 订阅 > 一次性（可持续） |
| **核心资源** | 需要什么能力？ | 用 AI 放大个人能力 |
| **关键业务** | 必须做什么？ | 聚焦最小可行产品 |
| **成本结构** | 投入多少钱？ | 控制固定成本 |

### 1.2 一人公司商业模式类型

| 模式 | 特点 | 适用场景 | 一人公司难度 |
|------|------|----------|--------------|
| **软件/SaaS** | 边际成本低，可规模化 | AI 工具、效率软件 | ⭐⭐⭐⭐ 需要技术能力 |
| **知识付费** | 成本低，毛利高 | 课程、咨询、社群 | ⭐⭐⭐ 内容生产能力 |
| **服务/咨询** | 交付重，收入可预期 | 企业服务、专业咨询 | ⭐⭐⭐ 获客能力 |
| **内容/媒体** | 积累粉丝，长尾收益 | 公众号、小红书、B站 | ⭐⭐⭐ 持续输出能力 |
| **电商/带货** | 现金流好，但库存风险 | 跨境电商、独立站 | ⭐⭐⭐⭐ 供应链能力 |
| **联盟/代理** | 轻资产，但依赖平台 | 工具推荐、服务分销 | ⭐⭐ 门槛低，利润薄 |

## 二、收费模式设计

### 2.1 常见定价模型

| 模型 | 说明 | 适用场景 | 一人公司适配 |
|------|------|----------|--------------|
| **一次性付费** | 买断产品 | 软件、模板、素材 | ⭐⭐⭐ 现金流好，但复购低 |
| **订阅制** | 按时付费 | SaaS、工具、资讯 | ⭐⭐⭐⭐ 稳定收入，需持续价值 |
| **会员制** | 享受权益 | 社群、资料库 | ⭐⭐⭐⭐ 轻交付，强复购 |
| **免费+增值** | 基础免费，高级付费 | 工具类产品 | ⭐⭐⭐ 获客容易，转化难 |
| **咨询/服务** | 按时间收费 | 专业服务 | ⭐⭐⭐ 交付重，难放大 |
| **广告/联盟** | 流量变现 | 内容平台 | ⭐⭐ 被动收入，但不稳定 |

### 2.2 定价策略

**一人公司定价原则**：

1. **高价策略**（相对低价更安全）
   - 高价筛选高质量用户
   - 高价覆盖获客成本
   - 高价才有利润空间

2. **锚定定价**
   - 设置 3 档：入门/标准/高级
   - 中间档为主推

3. **测试定价**
   - 初始定价可以偏低
   - 验证付费意愿后逐步涨价

### 2.3 一人公司价格参考

| 类别 | 价格区间 | 说明 |
|------|----------|------|
| 提示词/模板 | ¥9.9-99 | 低价走量 |
| 小课程 | ¥99-399 | 主流价格 |
| 大课程/训练营 | ¥499-1999 | 需要信任背书 |
| 社群年费 | ¥365-999 | 轻交付，高复购 |
| 咨询/小时 | ¥300-2000 | 看专业度 |
| SaaS 工具/月 | ¥19-199 | 看工具价值 |

## 三、毛利与成本

### 3.1 毛利计算

```
收入 - 成本 = 毛利
毛利 / 收入 = 毛利率
```

### 3.2 一人公司成本结构

| 成本项 | 说明 | 控制建议 |
|--------|------|----------|
| **获客成本** | 花钱买流量 | 初期用内容换流量 |
| **交付成本** | 产品/服务成本 | 边际成本趋近于 0 是目标 |
| **工具成本** | 软件、AI、服务器 | 选择性价比高的 |
| **时间成本** | 自己的时间 | 用 AI 放大 |
| **财务成本** | 税费、通道费 | 预留 10-15% |

### 3.3 盈亏平衡点

**必须计算**：
- 每月固定支出是多少？
- 每单毛利是多少？
- 需要多少单才能覆盖成本？

**示例**：
- 固定支出：¥5000/月
- 单均毛利：¥200
- 盈亏平衡：25 单/月

## 四、交付设计

### 4.1 交付边界

**必须明确**：
- 交付什么？
- 不交付什么？
- 交付形式是什么？
- 交付周期是多久？
- 售后政策是什么？

### 4.2 一人公司交付原则

1. **产品化 > 服务化**
   - 尽量把服务变成产品
   - 用产品代替人工交付

2. **标准化 > 定制化**
   - 标准化降低交付成本
   - 定制化只针对高客单价

3. **自动化**
   - 用 AI 自动回答常见问题
   - 用工具自动化交付流程

## 五、复购与停损

### 5.1 复购设计

**提升复购的方式**：
- 订阅制（自动续费）
- 会员制（持续价值）
- 迭代产品（持续提供新价值）
- 社群粘性（持续连接）

### 5.2 停损线

**一人公司必须设停损线**：

| 问题 | 答案 |
|------|------|
| 投入多少钱后认赔？ | 时间和金钱两个维度 |
| 多长时间没进展就放弃？ | 3 个月 / 6 个月 |
| 什么信号出现说明方向错了？ | 明确 2-3 个失败信号 |

**示例**：
- 投入超过 ¥2 万
- 3 个月内没有付费用户
- 目标用户反馈"不需要"
